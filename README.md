# PR Tracker — Node.js / Express

Reads PR data from `Final_Estimation.xlsx → All Handson WorkShop-May-Sep` and exposes
a full CRUD REST API for `PR_Details.json` and `Prod_Releases.json`.  
Lookup references come from `Module_Pages.json`, `Team.json`, and `Release_Timeline.json`.

---

## Setup

```bash
npm install
npm start          # server runs on http://localhost:3000
```

Put your data files inside the `data/` folder:

```
data/
  Final_Estimation.xlsx   ← source of truth
  PR_Details.json         ← auto-generated by sync
  Prod_Releases.json      ← auto-generated by sync
  Module_Pages.json       ← lookup reference
  Team.json               ← lookup reference
  Release_Timeline.json   ← lookup reference
```

## Column Mapping (Excel → JSON)

| Excel Column            | JSON Field             |
|-------------------------|------------------------|
| Module                  | Module                 |
| Modules/ Pages          | Page (array)           |
| Sprint                  | Dev_Sprint             |
| POC                     | Developer              |
| Status                  | Status                 |
| PR Raised Date          | PR Raised Date         |
| First PR Response Date  | PR First Response Date |
| PR Approval Date        | PR Approved Date       |
| PR Merged Date          | PR Merged Date         |
| Dependency PR Number    | Dependent_PRs (array)  |
| Planned Release Date    | Target_Release         |

All dates are formatted as `mm/dd/yyyy`.

## REST API

### Sync

| Method | Endpoint            | Description                                  |
|--------|---------------------|----------------------------------------------|
| POST   | `/api/sync/excel`   | Sync from bundled `Final_Estimation.xlsx`     |
| POST   | `/api/sync/upload`  | Upload new `.xlsx` file and sync (multipart)  |
| GET    | `/api/sync/preview` | Dry-run: parse Excel without writing files    |

### Pull Requests (`PR_Details.json`)

| Method | Endpoint                          | Description               |
|--------|-----------------------------------|---------------------------|
| GET    | `/api/prs`                        | List all PRs              |
| GET    | `/api/prs?module=Assessment`      | Filter by module          |
| GET    | `/api/prs?developer=Vijay`        | Filter by developer       |
| GET    | `/api/prs?status=TCR`             | Filter by status          |
| GET    | `/api/prs/:prNumber`              | Get single PR             |
| POST   | `/api/prs`                        | Add new PR                |
| PUT    | `/api/prs/:prNumber`              | Update PR (partial ok)    |
| DELETE | `/api/prs/:prNumber`              | Delete PR                 |
| POST   | `/api/prs/:prNumber/comments`     | Add comment to PR         |

### Prod Releases (`Prod_Releases.json`)

| Method | Endpoint                    | Description         |
|--------|-----------------------------|---------------------|
| GET    | `/api/releases`             | List all releases   |
| GET    | `/api/releases/:number`     | Get single release  |
| POST   | `/api/releases`             | Add / upsert        |
| PUT    | `/api/releases/:number`     | Update release      |
| DELETE | `/api/releases/:number`     | Delete release      |
| GET    | `/api/releases/timeline/all`| Full timeline ref   |

### Lookup

| Method | Endpoint                              | Description              |
|--------|---------------------------------------|--------------------------|
| GET    | `/api/lookup/modules`                 | All module names         |
| GET    | `/api/lookup/modules/:name/pages`     | Pages for a module       |
| GET    | `/api/lookup/developers`              | Developer list           |
| GET    | `/api/lookup/reviewers`               | PR Reviewer list         |
| GET    | `/api/lookup/team`                    | Full Team.json           |
| GET    | `/api/lookup/timeline`                | Release_Timeline.json    |
| GET    | `/api/lookup/module-pages`            | Full Module_Pages.json   |
